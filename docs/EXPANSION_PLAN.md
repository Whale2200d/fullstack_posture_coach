## Posture Coach – 확장 계획 (Commit 9)

MVP(크로스 트레이닝 – 스쿼트/데드리프트 중심) 이후,  
**요가/일반 웨이트 등 다른 운동과 개인화 기능**을 어떻게 확장할지에 대한 Phase 2+ 계획입니다.

---

## 1. Phase 1 (현재) 요약

- 운동 범위: 스쿼트, 데드리프트 등 소수의 크로스 트레이닝 동작.
- 모델: MediaPipe Pose + 규칙 기반 피드백.
- 개인화: 신체 정보(키/몸무게/연령)를 수집하지만, 단순 가중치/허용 범위 조정 수준.
- 코치 피드백: 좋아요/싫어요 + 1~5점 슬라이더, 코멘트.
- 재학습: ML 재학습 플로우 정의(문서 단계) + 라벨 데이터 구조 준비.

---

## 2. Phase 2 – 운동 종류 확장 (요가/웨이트)

### 2.1 목표

- 현재 구조를 크게 바꾸지 않고, **운동 타입만 추가**해서 기능을 확장.
- 요가(정적인 포즈)와 일반 웨이트(벤치프레스, 오버헤드 프레스 등)까지 확장 가능한 모델/데이터 구조 유지.

### 2.2 설계 원칙

1. **운동 타입은 enum이 아니라 데이터로 관리**
   - `docs/DB_SCHEMA_FIRESTORE.md` 의 `exercise_types` 컬렉션을 확장해,
     새 운동을 추가하는 작업을 **코드 수정이 아닌 데이터 추가**로 처리하는 것이 이상적.
2. **공통 포즈 엔진 + 운동별 규칙**
   - 포즈 추정(MediaPipe/YOLO)은 공통.
   - 운동별로 “관심 관절/각도/타이밍” 규칙만 분리해 구성.
3. **레이블/재학습 파이프라인 재사용**
   - `ml_labels` 컬렉션에 `exerciseTypeId` 가 이미 포함되어 있으므로,
     요가/웨이트 레이블도 동일 파이프라인으로 수집/학습.

### 2.3 작업 항목 (예시)

- `exercise_types` 에 다음과 같은 타입 추가:
  - `yoga_warrior2`, `yoga_tree`, `bench_press`, `overhead_press` 등.
- 운동별 기본 설정:
  - 타겟 관절 리스트, 올바른 각도 범위, 주요 에러 패턴 키워드(`knees_in`, `back_round`, `shoulder_elevated` 등).
- UI:
  - 홈 화면의 “오늘의 운동” 카드 목록에 새 운동 타입 표시.
  - 세션 히스토리/필터에서 운동 타입별 조회 지원.

## 3. Phase 2 – 개인화 고도화

### 3.1 목표

- 같은 운동이라도 **사용자 신체/레벨에 맞는 기준**을 제시.
- 예:  
  - 초보자는 깊이보다 안전을 우선 → 스쿼트 깊이 기준 완화.  
  - 무릎/허리 이슈가 있는 사용자는 특정 범위로 제한.

### 3.2 데이터 활용

- `users.profile`:
  - 키/몸무게/연령/성별/레벨 정보를 개인화 인자로 사용.
- `sessions.analysis.summary`:
  - 시간에 따른 점수/이슈 패턴을 추적해 **개인의 개선 추이**를 모델에 반영.
- `ml_labels.features`:
  - 동일 운동·비슷한 체형·비슷한 레벨 그룹의 피드백을 묶어,  
    **군집별(Cluster별) 기준값**을 학습.

### 3.3 구현 아이디어

1. **규칙 기반 보정 레이어**
   - 각 운동별 기본 각도 범위에 다음 보정을 적용:
     - 키가 큰 사용자: 상체 기울기 허용 범위 상향.
     - 초보자: “완벽한 깊이” 기준 대신 “안전한 깊이” 기준을 사용.
2. **ML 기반 개인화**
   - `features + profile` 을 입력으로, 코치 라벨(좋아요/점수)을 타깃으로 학습.
   - 출력: “이 사용자의 현재 운동 레벨에서 허용 가능한 자세 범위”.
   - 추론 시, 이 범위를 기반으로 에러 여부/강도를 결정.

## 4. Phase 3 – 모델/플랫폼 확장

### 4.1 YOLOv8-Pose 검증 및 통합 (별도 브랜치)

- `docs/ML_MODEL_COMPARISON.md` 에 정의된 계획대로:
  - YOLOv8-Pose + TFLite 변환 → `tflite_flutter` 등으로 Flutter 통합.
  - MediaPipe 대비 속도/정확도/메모리 사용량 비교.
  - 단체 수업/다중 인물 상황에서만 YOLO 기반 파이프라인을 사용하도록 조건부 적용.

### 4.2 플랫폼 확장

- 웹 코치 대시보드를 Flutter Web + Firebase Auth/Firestore로 구현.
- (선택) 관리자용 별도 웹 페이지에서 ML 상태, 재학습 로그, 모델 버전 관리 화면 추가.

---

## 5. 마일스톤 제안

1. **M1 – Cross Training MVP**
   - 스쿼트/데드리프트 + 기본 MediaPipe 분석 + 코치 레이블링 + Firebase 연동.
2. **M2 – 요가/웨이트 종목 추가**
   - `exercise_types` 확장, UI/분석 규칙 추가, 동일 ML 파이프라인 사용.
3. **M3 – 개인화 고도화**
   - 규칙 기반 보정 + ML 기반 개인화 모델 1차 적용.
4. **M4 – YOLOv8-Pose / 멀티 인물 실험**
   - 별도 브랜치에서 성능 검증 후, 필요 시 특정 시나리오에 통합.
