## Posture Coach – UI 와이어프레임 (Commit 5)

UI는 **MVP 기준**으로 “필수 화면 + 최소 동선”만 담았습니다.  
실제 디자인(Figma 등)을 만들 때 이 구조를 기준으로 확장하면 됩니다.

---

## 1. 전체 화면 목록

### 1. 사용자 앱 (모바일)

1. 온보딩 / 동의 화면
2. 로그인 / 회원가입
3. 홈 (운동 선택 + 최근 세션 요약)
4. 카메라 촬영 화면
5. 분석 결과 화면 (오버레이 + 텍스트/음성 피드백)
6. 세션 히스토리 / 상세
7. 프로필 / 신체 정보 입력

### 2. 코치 / 관리자 대시보드 (웹 중심)

1. 로그인 화면
2. 사용자 목록 화면
3. 세션/영상 목록 화면
4. 자세 리뷰 & 피드백 입력 화면
5. (추후) ML 상태/라벨 통계 화면

---

## 2. 사용자 플로우 – 화면 와이어프레임

### 2.1 로그인/온보딩

```text
[온보딩 1]          [동의 화면]                   [로그인 / 회원가입]
 ┌────────────┐     ┌──────────────────────┐      ┌──────────────────────┐
 |  앱 소개   |     |  • 영상 촬영/저장 동의 |      |  email / password    |
 |  (간단 문구)|    |  • Firebase 저장 안내 |      |  [로그인] [회원가입]   |
 |            |     |  [동의]  [동의 안함]  |      └──────────────────────┘
 └────────────┘     └──────────────────────┘
```

### 2.2 홈 화면

```text
[홈]

│  상단: "Posture Coach" 로고/타이틀  │
│────────────────────────────────────│
│  카드 1: 오늘의 운동               │
│   - [스쿼트]  [데드리프트]         │
│────────────────────────────────────│
│  버튼: [촬영 시작]                  │

│   - 2026-03-10 스쿼트 8세트        │
│   - 2026-03-09 데드리프트 5세트    │
```

### 2.3 카메라 촬영 화면

```text
[카메라 촬영 화면]
┌────────────────────────────────────┐
│  상단: 선택된 운동명 (예: 스쿼트)  │
│────────────────────────────────────│
│        (카메라 프리뷰 영역)        │
│    전체 화면에 사용자 전신이 보이도록│
│    가이드 라인(실루엣) 표시        │
│────────────────────────────────────│
│  좌: [세션 시간 표시]  00:12       │
│  우: [반대 카메라 전환 아이콘]     │
│────────────────────────────────────│
│  중앙 하단: ● 촬영 버튼           │
│   - 짧게 탭: 영상 촬영 시작/정지   │
│   - 녹화 중일 때: 빨간 점 + 타이머 │
└────────────────────────────────────┘
```

### 2.4 분석 결과 화면 (오버레이 + 피드백)

```text
[분석 결과]
┌────────────────────────────────────┐
│ 상단 바:  "스쿼트 분석 결과"        │
│────────────────────────────────────│
│  영상 재생 영역 + 자세 오버레이     │
│  - 관절(랜드마크) 점/선 표시        │
│  - 잘못된 구간: 빨간 선/텍스트     │
│────────────────────────────────────│
│  하단 패널                         │
│  - 텍스트 피드백 목록               │
│    • "허리를 더 펴세요."           │
│    • "무릎이 너무 앞으로 나갔어요." │
│  - [다시 보기]  [새로 촬영]         │
│  - (옵션) [음성 피드백 재생]        │

```

### 2.5 세션 히스토리 & 프로필

```text
[세션 히스토리 탭]
┌────────────────────────────────────┐
│  날짜별 리스트                     │
│   - 2026-03-10 스쿼트 8세트 (상세) │
│   - 2026-03-09 데드리프트 5세트    │
│  각 항목 터치 시:                  │
│   - 영상 재생 + 분석 요약          │

┌────────────────────────────────────┐

│  신체 정보 입력 폼                 │
│   - 키, 몸무게, 연령, 운동 레벨   │
│  [저장] 버튼                       │
└────────────────────────────────────┘
```


## 4. 확장 포인트 메모

- 운동 타입(스쿼트/데드리프트/요가 등)은 홈 화면의 “오늘의 운동” 카드와 카메라 상단 운동명에서만 바꿔주면 되도록 설계.
- ML/재학습 상태 화면은 초기에는 단순 텍스트/표(모델 버전, 정확도, 최근 재학습 시간)로 시작해, 추후 차트/그래프로 확장.
- 반응형 디자인을 고려해 모바일/태블릿/웹에서 공통 컴포넌트 구조를 재사용할 수 있도록 Flutter 위젯을 설계한다.
