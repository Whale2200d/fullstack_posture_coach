## Posture Coach – 요구사항 정의서 (Commit 4)

---

### 1. 개요

- **목적**
  - 카메라 기반 **자세 분석**으로 크로스 트레이닝(스쿼트, 데드리프트 등)을 교정하고,
    이후 요가/일반 웨이트 등 다양한 운동으로 확장 가능한 **자세 코칭 플랫폼**을 만든다.
- **대상 플랫폼**
  - 모바일: iOS, Android
  - 웹/데스크톱: Chrome(Web), macOS(Desktop)
- **기술 스택**
  - 프론트엔드: Flutter
  - 백엔드: Firebase (Auth, Firestore, Storage, Cloud Functions)
  - ML: MediaPipe Pose → 추후 YOLOv8 Pose 옵션, Cloud Functions 기반 재학습 루프

---

### 2. 주요 사용자 및 역할

1. **일반 사용자 (운동자)**
   - 본인 운동(스쿼트, 데드리프트 등)을 촬영하고 실시간/녹화 기반으로 **자세 피드백**을 받는다.
   - 신체 정보(키, 몸무게, 연령 등)를 입력해 **개인화된 코칭**을 받는다.

2. **코치 / 관리자**
   - 대시보드에서 사용자 영상/자세 분석 결과를 검토한다.
   - 자세가 좋은지/나쁜지, 얼마나 나쁜지에 대한 **레이블(좋아요/싫어요 + 1~5점)** 을 입력해 ML 학습 데이터를 쌓는다.

3. **시스템 / ML 파이프라인**
   - MediaPipe Pose로 랜드마크를 추출하고, 표준 자세와 비교해 **기본 규칙 기반 피드백**을 제공한다.
   - 코치가 입력한 피드백을 모아 **Cloud Functions에서 주기적으로 재학습**을 수행한다.
   - 인식률이 일정 기준(예: 85%) 이상일 때, 잘못된 자세를 **자동으로 코칭**한다.

---

### 3. 기능 요구사항

#### 3.1 사용자 측 기능

1. **회원가입 / 로그인**
   - 이메일·비밀번호 기반 Firebase Auth 로그인.
   - (추후) SNS 로그인 옵션 확장 가능.

2. **신체 정보 입력/수정**
   - 키, 몸무게, 연령, 성별(선택), 운동 레벨(초급/중급/고급) 등 입력.
   - 정보 변경 시 이후 분석에 반영.

3. **카메라 촬영 및 재생**
   - 실시간 카메라 프리뷰 제공.
   - 운동 세션 단위로 동영상 촬영 및 로컬 저장.
   - 촬영 후 영상 재생 UI 제공.

4. **자세 분석 및 피드백**
   - 촬영된 영상 또는 실시간 프레임을 MediaPipe Pose에 전달해 랜드마크(관절 좌표)를 추출.
   - 무릎/허리/어깨 등 주요 관절 각도를 계산해 **표준 자세와 비교**.
   - 잘못된 구간은 화면에 **빨간 선/하이라이트**로 표시.
   - 텍스트 피드백(예: “허리를 더 펴세요”, “무릎이 앞으로 나갔습니다”)을 즉시 표시.
   - (옵션) TTS를 통해 음성 피드백 제공.

5. **세션 기록**
   - 사용자별로 **운동 종류, 날짜/시간, 분석 결과 요약**을 Firestore에 저장.
   - 최근 세션 리스트 및 요약 통계(횟수, 개선도 등)를 조회 가능.

#### 3.2 코치 / 관리자 측 기능

1. **대시보드**
   - 사용자 목록, 최근 업로드된 영상/세션 리스트 조회.
   - 특정 사용자의 자세 분석 결과(랜드마크/각도/에러 포인트) 시각화.

2. **피드백 레이블링**
   - 영상/프레임 단위로 **좋아요 / 싫어요** 버튼 제공.
   - 잘못된 정도를 **1~5점 슬라이더**로 입력.
   - 입력된 피드백은 Firestore에 ML 학습용 라벨 데이터로 저장.

3. **ML 상태 모니터링 (추후)**
   - 현재 모델 버전, 최근 재학습 시점, 검증 정확도(예: 85% 이상 여부) 등의 상태를 조회.

#### 3.3 ML 개인화 기능

1. **개인별 보정**
   - 신체 정보(키, 몸무게, 연령 등)에 따라 허용 각도 범위나 기준을 조정.
   - 예: 키가 큰 사용자는 스쿼트 시 상체 기울기를 다르게 허용.

2. **재학습 데이터 누적**
   - 코치 레이블(좋아요/싫어요 + 1~5점)을 사용자/운동 종류별로 저장.
   - 충분한 데이터가 쌓이면 Cloud Functions에서 주기적으로 재학습.

3. **모델 선택 / 교체**
   - 기본은 MediaPipe Pose 기반 분석.
   - 별도 브랜치에서 YOLOv8 Pose를 시험하고, 속도/정확도 벤치마크 후 필요 시 통합.

### 4. 비기능 요구사항

1. **성능**
   - 모바일 기기에서 **실시간 분석 시 최소 24fps**, 목표 30fps.
   - 한 세션 업로드/분석이 사용자가 느끼기에 “버벅임이 없다”고 느낄 정도의 응답성(수초 이내).

2. **가용성 / 안정성**
   - 네트워크가 불안정할 때도 **오프라인 모드**로 촬영 및 로컬 저장 가능.
   - 연결 복구 시 Firebase Storage/Firestore로 자동 업로드/동기화.

3. **보안 / 프라이버시**
   - 첫 촬영 전, **영상/이미지 저장 및 분석에 대한 동의 화면** 필수.
   - Firebase Auth로 인증된 사용자만 자신의 영상/데이터에 접근 가능.
   - 동영상/이미지 파일은 Firebase Storage에 저장하고, 공용 URL 노출은 최소화.

4. **확장성**
   - 운동 종류를 enum/모듈 단위로 관리해, 요가/일반 웨이트 종목을 손쉽게 추가.
   - ML 모듈은 특정 운동 타입에 종속되지 않고, **운동 타입별 구성 파일/파이프라인**을 분리.

5. **유지보수성**
   - Flutter 모듈 구조: 카메라, 자세 분석, 피드백 UI, Firebase 연동, ML 연동 등으로 분리.
   - 문서는 `docs/` 폴더에 커밋별로 관리 (환경 세팅, Firebase, 요구사항 등).

---

### 5. 플랫폼 / 환경 요구사항

1. **Flutter**
   - 버전: Flutter stable 3.41.x, Dart 3.11.x 기준.
   - 최소 SDK 범위: `>=3.0.0 <4.0.0` (pubspec.yaml 기준).

2. **Firebase**
   - Firebase 프로젝트 이름/ID에는 **`crossfit` 단어를 사용하지 않는다.**
   - 사용 서비스: Auth, Firestore, Storage, (추후) Cloud Functions, Messaging, Analytics 등.

3. **지원 OS**
   - iOS: 최신 2~3개 메이저 버전 (예: iOS 17, 18 이상).
   - Android: minSdkVersion(Flutter 기본) 이상, 최신 릴리스까지.
   - Web: Chrome 최신 버전.
   - 데스크톱: macOS 최신 메이저 버전 기준.

4. **개발 환경**
   - IDE: VSCode / Cursor (Flutter, Dart 확장 설치).
   - iOS 개발용: Xcode, iOS Simulator.
   - Android 개발용: Android Studio, Android Emulator.
